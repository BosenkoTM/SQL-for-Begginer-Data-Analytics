# Лекция 4. Взаимодействие Python и PostgreSQL (Psycopg2, SQLAlchemy)

## Цель занятия
Изучить иерархию инструментов для работы с реляционными базами данных в Python: от низкоуровневого драйвера до объектно-реляционного отображения (ORM), и научиться выбирать подходящий инструмент под аналитические задачи.

## Основные задачи
1.  **Работа с драйвером (Low-Level):**
    *   Установить соединение с PostgreSQL используя библиотеку `psycopg2`.
    *   Изучить работу с курсорами (`cursor`) для выполнения «сырых» SQL-запросов.
    *   Научиться применять параметризованные запросы для защиты от SQL-инъекций.

2.  **Интеграция с Pandas (Mid-Level):**
    *   Настроить `SQLAlchemy Engine` для управления пулом соединений.
    *   Освоить метод `pd.read_sql()` для прямой выгрузки результатов SQL-запросов в DataFrame (основной сценарий аналитика).

3.  **Введение в ORM (High-Level):**
    *   Разобрать концепцию Object-Relational Mapping (ORM).
    *   Описать таблицы базы данных как классы Python (Models).
    *   Выполнить выборку и фильтрацию данных, оперируя объектами и методами Python, а не SQL-кодом.

## Инструментарий
*   **Python 3.x**
*   **Psycopg2** (DB-API драйвер)
*   **SQLAlchemy** (Core + ORM)
*   **Pandas** (Анализ данных)
